# 响应式混合导航布局任务背景

## 任务时间
2026-01-25

## 任务目标
实现响应式混合导航布局：
- **PC 端 (Desktop)**: 保持顶部导航，完整菜单
- **移动端 (Mobile)**: 切换为底部导航栏 (Dock/TabBar)

## 设计理念

### 为什么采用混合布局？
1. **人体工学考虑**
   - PC: 鼠标移动到顶部很轻松
   - Mobile: 拇指热区，单手操作友好

2. **视觉层级**
   - PC: 符合网页浏览习惯（F型浏览模式）
   - Mobile: 类似原生 App 体验

3. **空间占用**
   - PC: 顶部导航占据横向空间，适合展示复杂菜单
   - Mobile: 底部导航只能放 3-5 个核心功能

### 核心设计原则

#### 顶部导航 (Header)
- **手机端**: 只显示 Logo 和头像
- **PC端 (md:)**: 显示搜索框、按钮组、完整菜单

#### 底部导航 (Dock)
- **手机端**: 固定在底部，宽度 100%，类似原生 App TabBar
- **PC端 (md:)**: 悬浮在底部中间，圆角胶囊状，或作为快捷工具

## 技术实现要点

### 1. Tailwind CSS 媒体查询
- 使用 `hidden md:flex` 控制元素显示
- 使用 `md:`, `lg:` 等断点实现响应式

### 2. iPhone 安全区域适配
```css
.safe-area-bottom {
  padding-bottom: env(safe-area-inset-bottom);
}
```

### 3. 移动端浏览器工具栏冲突
- 使用"胶囊式悬浮"设计
- 距离底部约 20px，避开浏览器 UI
